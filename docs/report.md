# Flappy Bird 强化学习实验报告

## 一、实验目的

本实验的目标是基于 Q-learning 方法训练一个能够自动游玩 Flappy Bird 的智能体。智能体需要根据游戏环境返回的观测值判断当前状态，并在“不拍翅”和“拍翅”两个动作之间进行选择，使小鸟尽量通过更多管道、获得更高分数。

本实验主要完成以下工作：

1. 补全 `GameAI` 类中的 Q-learning 相关代码，使训练、评估和模型保存流程能够正常运行。
2. 设计并比较不同的状态表示方式，观察 `process_obs()` 对训练效果的影响。
3. 处理实验结果不可重复的问题，固定训练和评估中的随机种子，并在后续实验中使用多随机种子比较。
4. 调整学习率 $\alpha$、探索率 $\epsilon$、折扣因子 $\gamma$、死亡惩罚和衰减策略，观察这些因素对模型表现的影响。

## 二、实验原理

讲义中已经给出了强化学习和 Q-learning 的基本公式，因此本节只结合本实验实现作简要说明。

### 1. Flappy Bird 的强化学习建模

在本实验中，Flappy Bird 可以看作一个序列决策问题。每一帧中，智能体根据当前观测状态 $s$ 选择动作 $a$，环境返回新的状态 $s'$ 和奖励 $r$。动作空间较简单：

- `0`：不拍翅；
- `1`：拍翅。

环境返回的原始观测 `obs` 包含管道位置、小鸟位置、小鸟速度等信息。由于本实验采用表格式 Q-learning，不能直接使用连续观测值作为 Q 表索引，所以需要通过 `process_obs()` 将观测值离散化为整数元组。

### 2. Q 表与状态表示

代码中使用字典 `self.q` 保存 Q 表，其键为 `(state, action)`，值为对应的 Q 值。如果某个状态-动作对尚未出现，则默认 Q 值为 0。

基础状态表示为：

$$
s=(x_{\text{pipe}}, y_{\text{bottom}}, v_y)
$$

其中 $x_{\text{pipe}}$ 表示小鸟与最近管道的水平距离，$y_{\text{bottom}}$ 表示小鸟与下方管道的竖直相对位置，$v_y$ 表示小鸟竖直速度。代码中使用 `obs_mul_factor` 对连续观测值放大并取整，使状态能够作为 Q 表的键。

这种状态设计保留了避障中比较关键的信息，但也存在近似误差：如果离散化过粗，不同真实位置可能被合并到同一个状态；如果离散化过细，状态数量会变多，训练中很多状态难以充分访问。

### 3. Q-learning 更新与动作选择

每次游戏交互得到 $(s,a,r,s')$ 后，程序使用 Q-learning 公式更新 Q 表：

$$
Q(s,a)\leftarrow (1-\alpha)Q(s,a)+\alpha\left[r+\gamma\max_{a'}Q(s',a')\right]
$$

其中 $\alpha$ 是学习率，控制新样本对旧 Q 值的影响；$\gamma$ 是折扣因子，控制未来奖励的重要程度。

训练时使用 $\epsilon$-greedy 策略选择动作：以 $\epsilon$ 的概率随机选择动作，用于探索；以 $1-\epsilon$ 的概率选择当前 Q 值较高的动作，用于利用已有经验。评估模型时关闭随机探索，只选择当前 Q 表中价值更高的动作。

### 4. 奖励与超参数

环境原始奖励包括存活奖励、通过管道奖励和死亡惩罚。本实验代码中对死亡惩罚进行了放大，使智能体更明确地学习到“死亡状态需要避免”。除奖励外，实验中重点关注以下超参数：

- 学习率 $\alpha$：影响 Q 值更新幅度；
- 探索率 $\epsilon$：影响随机探索比例；
- 折扣因子 $\gamma$：影响未来奖励在当前决策中的权重；
- 训练轮数 `iteration`：影响智能体经历的游戏局数；
- 状态离散化系数 `obs_mul_factor`：影响状态空间大小。

此外，固定的 $\epsilon$ 和固定的 $\alpha$ 不一定适合整个训练过程。训练早期通常需要更多探索，训练后期则更需要稳定利用已有策略；学习率也类似，后期如果仍保持较大步长，Q 值可能持续震荡。因此本实验还比较了探索率衰减和学习率衰减。

## 三、实验过程

### 1. 补全代码与初始调试

首先根据作业框架补全 `src/q_learning.py` 中的主要函数，包括：

1. `get_q_value()`：读取指定状态-动作对的 Q 值；
2. `best_future_reward()`：计算下一状态下可取得的最大 Q 值；
3. `update()`：根据 Q-learning 公式更新 Q 表；
4. `choose_action()`：按照 $\epsilon$-greedy 策略选择动作；
5. `process_obs()`：将环境观测转换为离散状态。

调试过程中遇到了两个和 Python 数据结构有关的问题，已记录在 `docs/konwledge.md` 中：

| 问题 | 原因 | 处理方式 |
| :--- | :--- | :--- |
| `TypeError: unhashable type: 'list'` | `state` 如果使用 list，不能作为字典键的一部分 | 在 `process_obs()` 末尾返回 `tuple(state)` |
| `TypeError: 'set' object is not subscriptable` | `available_actions()` 返回 set，set 不能用下标访问 | 随机探索时使用 `random.choice(tuple(...))` |

这两个问题虽然不是算法本身的问题，但会影响 Q 表索引和动作采样。修复后，训练流程可以正常运行，并能按设定间隔输出平均得分。

### 2. 状态表示实验

完成基础训练流程后，首先尝试修改 `process_obs()` 的状态设计，实验记录保存在 `experiments/change_obs` 中。

| 实验 | 修改内容 | 最终评估平均分 |
| :--- | :--- | ---: |
| `baseline` | 使用基础状态表示 | 30.26 |
| `change_x` | 调整水平距离的计算方式，改为小鸟左端到管道左端的距离 | 13.00 |
| `change_y` | 调整竖直距离，改为小鸟底端到下方管道的距离 | 19.50 |
| `change_and_add_y` | 在调整竖直距离的基础上，加入小鸟到上方管道的相对距离 | 14.10 |

从结果看，这几组状态修改没有带来提升。主要原因可能是这些修改多数只是对已有观测量进行平移、换参考点或增加相近变量，并没有引入新的有效信息。对于表格式 Q-learning 来说，状态只是 Q 表索引，如果只是对原变量做线性变换，智能体可利用的信息并没有明显增加。

另外，`change_and_add_y` 虽然增加了一个状态维度，但状态空间也随之增大。在训练轮数不变的情况下，每个状态被访问的次数可能下降，因此学习效果反而变差。这个阶段说明，状态设计不应只改变数值形式，更重要的是减少状态混淆或加入真正有助于判断管道缺口位置的信息。

### 3. 固定随机种子

在早期实验中，同一组参数多次运行得到的分数差异较大。分析后发现，训练过程中存在多处随机性，包括环境初始化、管道生成、$\epsilon$-greedy 随机探索、Python 随机数和 NumPy 随机数。

因此后续加入了随机种子设置，对应提交为 `feat: set random seeds for repeatable experiments`。主要修改包括：

1. 在训练脚本中设置 `random.seed(42)` 和 `np.random.seed(42)`；
2. 训练环境初始化时使用固定 seed；
3. 评估函数中使用 `seed=42+i`，使评估局面固定；
4. 增加 `--eval-episodes` 参数，并在日志中记录每次评估局数。

这一步先解决了“同一代码运行时无法对齐测试条件”的问题，使不同参数实验之间更容易比较。但后续实验仍发现，单一种子下的高分可能带有偶然性，因此后面又进一步改成每组参数使用多个随机种子进行比较。

### 4. 学习率和探索率实验

在状态表示实验效果不明显后，实验转向调整学习率 $\alpha$ 和探索率 $\epsilon$。这两个参数分别影响 Q 表更新速度和随机探索比例，对训练结果影响较直接。

#### 4.1 粗网格搜索

粗网格实验保存在 `experiments/tune_alpha_epsilon`。实验固定 $\gamma=0.95$、训练轮数 `50000`、评估局数 `100`，比较：

$$
\alpha \in \{0.1,0.2,0.3,0.5,0.7\}
$$

$$
\epsilon \in \{0.0,0.02,0.05,0.1\}
$$

按当前结果文件中最后一次训练记录统计，表现较好的几组如下：

| $\alpha$ | $\epsilon$ | 最终评估平均分 |
| ---: | ---: | ---: |
| 0.2 | 0.05 | 76.99 |
| 0.5 | 0.0 | 70.88 |
| 0.1 | 0.0 | 46.14 |
| 0.3 | 0.02 | 39.73 |
| 0.1 | 0.02 | 34.74 |

从粗网格结果看，较大的学习率并不一定更好。例如 $\alpha=0.7$ 的多数组合得分较低，说明 Q 值更新幅度过大时容易产生波动。探索率也不是越大越好，$\epsilon=0.1$ 的多数组合表现一般，可能是训练后期仍有较多随机动作，影响了已学到策略的稳定性。

#### 4.2 精细搜索

根据粗网格结果，后续在较小学习率和较低探索率附近继续搜索，结果保存在 `experiments/fine_tune_alpha_epsilon`。本阶段比较：

$$
\alpha \in \{0.05,0.08,0.10,0.12,0.15\}
$$

$$
\epsilon \in \{0.01,0.02,0.03\}
$$

当前表现较好的几组如下：

| $\alpha$ | $\epsilon$ | 最终评估平均分 |
| ---: | ---: | ---: |
| 0.08 | 0.02 | 112.06 |
| 0.12 | 0.02 | 92.89 |
| 0.08 | 0.01 | 64.40 |
| 0.15 | 0.03 | 63.94 |
| 0.10 | 0.01 | 49.86 |

精细搜索中，$\alpha=0.08,\epsilon=0.02$ 在当前记录中得分较高。这个结果说明，在本实验设置下，较小但不太小的学习率有助于稳定更新 Q 表；适度探索也有必要，因为完全不探索可能过早依赖当前 Q 表，而探索过多又会在训练中不断引入随机动作。

这一阶段还不能说明该组参数就是最终最优参数。后续仍需要结合 $\gamma$、探索率衰减、学习率衰减和 reward 调整继续比较。

### 5. 探索率衰减实验

在固定 $\epsilon$ 的实验中，模型表现仍不够理想。查阅相关经验后可以发现，探索率通常不会一直保持常量，而是会随着训练进行逐渐减小。原因是训练前期需要更多随机探索来覆盖不同状态，训练后期则应减少随机动作，使智能体更多利用已经学到的 Q 表。

因此在代码中实现了三种 $\epsilon$ 衰减方式，并与固定 $\epsilon$ 进行比较：

1. 线性衰减：按训练轮数线性下降到最小探索率；
2. 指数衰减：按指数函数逐渐下降；
3. 乘数衰减：每轮乘以固定衰减系数。

实验记录保存在 `experiments/epsilon_decay`。该阶段固定 $\alpha=0.1$、$\gamma=0.95$、训练轮数 `50000`、评估局数 `100`。主要结果如下：

| 实验 | 设置 | 最终评估平均分 |
| :--- | :--- | ---: |
| `baseline_0.02` | 固定 $\epsilon=0.02$ | 34.74 |
| `baseline_0.1` | 固定 $\epsilon=0.1$ | 9.48 |
| `linear_mid` | 线性衰减，$\epsilon_{\min}=0.01$ | 55.25 |
| `exp_fast` | 指数衰减，$\tau=5000$ | 13.94 |
| `exp_slow` | 指数衰减，$\tau=10000$ | 1356.81 |
| `mult_fast` | 乘数衰减，$d=0.9998$ | 52.47 |
| `mult_slow` | 乘数衰减，$d=0.99992$ | 63.88 |

从这个阶段看，衰减策略整体比固定较大探索率更合理。其中指数衰减在单种子实验中出现了最高分，因此后续实验选择指数衰减作为主要探索率策略。不过这个阶段也暴露出一个问题：`exp_slow` 的分数远高于其他组，可能包含较强随机性，不能只根据这一组单次结果判断最终最优。

### 6. 多随机种子实验设置

在探索率衰减实验后，仍然能看到结果波动较大的现象。即使固定了测试 seed，不同训练 seed 也会让智能体经历不同的探索轨迹，最终 Q 表可能差别很大。为了避免只根据单次训练做结论，后续增加了可配置随机种子的功能，并编写了通用多种子实验脚本。

后续多数组实验采用 3 个随机种子：

$$
\text{seed} \in \{42,100,2026\}
$$

`scripts/run_multiseed_experiments.sh` 会对同一组参数分别运行这 3 个 seed，并把结果保存到对应实验目录中。这样可以同时观察平均表现和种子之间的差异。后续 reward、$\gamma$、$\alpha$ 衰减实验都采用了这种多种子设置。

### 7. 死亡惩罚实验

在 reward 设计中，死亡惩罚会直接影响智能体对危险动作的估计。如果死亡惩罚过大，后期训练中一次失败可能导致 Q 值变化过剧烈，从而影响已有策略的稳定性；但如果惩罚过小，智能体又可能无法充分学习到死亡状态的严重性。

因此本实验比较了不同死亡惩罚，实验记录保存在 `experiments/change_rewards`。该阶段固定 $\alpha=0.1$、$\gamma=0.95$，使用指数探索率衰减，并对每组参数使用 3 个随机种子。

| 死亡惩罚 | Seed 42 | Seed 100 | Seed 2026 | 平均分 |
| ---: | ---: | ---: | ---: | ---: |
| -1000 | 48.09 | 39.40 | 40.63 | 42.71 |
| -500 | 41.48 | 52.08 | 29.85 | 41.14 |
| -100 | 2052.40 | 36.36 | 67.68 | 718.81 |
| -10 | 25.91 | 63.39 | 45.81 | 45.04 |

从结果看，`-100` 在 Seed 42 下出现了非常高的分数，但另外两个种子并没有同样高的表现。因此这一结果更像是某个训练轨迹下形成了较好的策略，而不是说明 `-100` 在所有情况下都稳定最优。除该异常高分外，多数组合的平均表现大致在几十分类别，说明 reward 调整能影响训练结果，但仍受随机种子和状态表示限制。

这个阶段的结论是：死亡惩罚不能简单地越大越好。过大的惩罚可能增加训练后期的波动，过小的惩罚又可能削弱死亡信号。本实验后续倾向于使用中等强度的死亡惩罚，并结合多种子结果判断。

### 8. 折扣因子实验

随后对折扣因子 $\gamma$ 进行了实验，结果保存在 `experiments/change_gamma`。$\gamma$ 决定智能体重视未来奖励的程度。对于 Flappy Bird 来说，智能体不能只看当前一帧是否安全，还需要提前调整高度以通过后面的管道，因此 $\gamma$ 过小可能不利于长远规划；但 $\gamma$ 过接近 1，也可能让未来估计带来的波动更明显。

实验先比较了 $\gamma=0.8,0.9,0.95,0.99,1.0$，之后又在较可能的区间内增加了 $0.96,0.97,0.98$。每组使用 3 个随机种子。主要结果如下：

| $\gamma$ | Ep 50k 平均分 | 40k-50k 末期平均分 |
| ---: | ---: | ---: |
| 1.00 | 21.88 | 19.66 |
| 0.99 | 70.74 | 62.03 |
| 0.98 | 73.42 | 76.11 |
| 0.97 | 113.42 | 93.15 |
| 0.96 | 64.97 | 55.11 |
| 0.95 | 718.81 | 294.10 |
| 0.90 | 51.09 | 34.54 |
| 0.80 | 9.24 | 13.22 |

其中 $\gamma=0.95$ 的最终平均分受到 Seed 42 的高分影响较大，不能只看最终均值。相比之下，$\gamma=0.97$ 和 $\gamma=0.98$ 的后期表现更平稳，尤其 $\gamma=0.97$ 在 40k 到 50k 区间的趋势较好。因此后续更倾向于在 $0.97$ 附近继续实验。

### 9. 学习率衰减实验与主线选择

上课时老师提到，表格式 Q-learning 在一定条件下是可以收敛的，其中学习率需要满足类似 Robbins-Monro 条件：

$$
\sum_t \alpha_t = \infty,\qquad \sum_t \alpha_t^2 < \infty
$$

但在前面的实验中，固定学习率下模型后期经常出现震荡，并没有明显收敛到稳定分数。为了观察学习率衰减是否能改善这个问题，本实验又实现并比较了三类 $\alpha$ 衰减方式：

1. 线性衰减：随训练轮数线性减小；
2. 指数衰减：随训练轮数按指数形式减小；
3. 计数衰减：根据每个状态-动作对的访问次数调整学习率。

实验记录保存在 `experiments/alpha_decay`。该阶段使用指数探索率衰减，并对每组参数使用 3 个随机种子。主要结果如下：

| 学习率策略 | Ep 50k 平均分 | 40k-50k 末期平均分 |
| :--- | ---: | ---: |
| 固定 $\alpha=0.1$ | 113.42 | 93.15 |
| 固定 $\alpha=0.5$ | 11.11 | 13.49 |
| 线性衰减，$\alpha_{\min}=0.01$ | 129.86 | 134.94 |
| 线性衰减，$\alpha_{\min}=0.05$ | 131.86 | 138.21 |
| 指数衰减，$\alpha_{\min}=0.01$ | 222.01 | 174.42 |
| 指数衰减，$\alpha_{\min}=0.05$ | 209.99 | 178.64 |
| 计数衰减，$p=1.0$ | 38.71 | 28.44 |
| 计数衰减，$p=0.6$ | 40.36 | 39.70 |
| 计数衰减，$p=0.4$ | 256.07 | 298.19 |
| 计数衰减，$p=0.4,\alpha_{\min}=0.05$ | 162.56 | 157.72 |

从结果看，学习率衰减确实能缓解固定高学习率带来的不稳定问题。指数衰减和线性衰减都比固定 $\alpha=0.5$ 好很多。计数衰减中，理论上更接近 Q-learning 收敛条件的 $p=1.0$ 和 $p=0.6$ 反而表现不理想，可能是学习率下降太快，导致很多状态还没有充分学习就更新变慢。$p=0.4$ 虽然不严格满足上述平方和收敛条件，但实验分数更高，说明在有限训练轮数和当前状态设计下，完全追求理论收敛不一定带来更好的游戏分数。

为了进一步比较学习率衰减速度，又补充了 `experiments/change_alpha` 实验。该实验一方面细化了指数衰减的时间常数 $\tau$ 和学习率下限，另一方面比较了计数衰减中不同幂次 $p$。补充实验的主要结果如下：

| 学习率策略 | Ep 50k 平均分 | 40k-50k 末期平均分 | 现象 |
| :--- | ---: | ---: | :--- |
| 指数衰减，$\tau=5000,\alpha_{\min}=0.05$ | 358.45 | 255.52 | 最终均分最高，但种子差异较大 |
| 指数衰减，$\tau=8000,\alpha_{\min}=0.05$ | 161.57 | 199.48 | 后期表现较好，但末尾有回落 |
| 指数衰减，$\tau=15000,\alpha_{\min}=0.01$ | 230.80 | 211.42 | 较慢衰减配低下限表现较好 |
| 计数衰减，$p=0.3$ | 118.32 | 124.75 | 衰减偏慢，后期扰动较大 |
| 计数衰减，$p=0.35$ | 89.71 | 105.58 | 整体表现一般 |
| 计数衰减，$p=0.4$ | 256.07 | 298.19 | 末期平均分最高 |
| 计数衰减，$p=0.45$ | 174.48 | 151.82 | 比 $p=0.4$ 更保守 |
| 计数衰减，$p=0.5$ | 95.02 | 85.80 | 衰减较快，学习不足 |

综合这些结果，本实验选择 **计数衰减 $p=0.4$** 作为报告主线。选择它的原因不是单纯因为某一次得分最高，而是因为它在机制和实验现象上都比较适合本任务。

首先，计数衰减是按每个 `(state, action)` 的访问次数调整学习率。频繁访问的状态会逐渐减小步长，使 Q 值更加稳定；较少访问的状态仍保持较大学习率，便于后续继续学习。Flappy Bird 的状态是离散化后的表格式状态，不同状态被访问的频率差异很大，因此这种局部学习率比全局按 episode 衰减更贴合 Q 表学习过程。

其次，从 $p$ 的对比可以看到一个比较清楚的趋势：$p=0.3$ 和 $p=0.35$ 衰减偏慢，后期仍然有较大更新扰动；$p=0.45$ 和 $p=0.5$ 衰减偏快，模型学习不足；$p=0.4$ 位于中间，末期平均分最高。虽然 $p=0.4$ 不严格满足理论收敛条件，但本实验训练轮数有限，目标也是获得较好的游戏分数，而不是证明无限步下的严格收敛。因此这里更重视有限训练时间内的实际表现。

需要说明的是，`count_p_0.4` 的结果仍然存在随机种子差异，不能理解为完全稳定收敛。但相比指数衰减，它更能作为本实验的主线：指数衰减是一个较好的工程对照，而计数衰减更能体现“根据状态-动作访问频率调整学习率”的 Q-learning 思路。

### 10. 状态离散化粒度与参数适配实验

在确定主要超参数方向后，本实验又进一步比较了 `obs_mul_factor` 对训练效果的影响。`obs_mul_factor` 控制连续观测值离散化的分箱粒度。它越大，状态划分越细，智能体能区分更细的位置差异；但同时 Q 表状态数量也会增加，每个状态被访问到的次数减少。

在当前三维状态表示下，如果只考虑分箱数量的近似变化，`obs_mul_factor` 从 30 增加到 40 时，状态空间规模约变为：

$$
\left(\frac{40}{30}\right)^3=\frac{64}{27}\approx 2.37
$$

也就是说，状态数量大约增大到原来的 2.37 倍。在训练轮数不变时，单个状态-动作对的平均访问次数会明显下降。因此，较大的 `obs_mul_factor` 不能直接沿用原来的训练参数，否则容易出现 Q 表稀疏、学习不足的问题。

首先，在其他参数基本不变、训练轮数仍为 50000 的情况下，比较了不同 `obs_mul_factor`：

| `obs_mul_factor` | 训练轮数 | $\epsilon$ 衰减时间常数 | 最终评估平均分 |
| ---: | ---: | ---: | ---: |
| 10 | 50000 | 10000 | 38.18 |
| 20 | 50000 | 10000 | 110.04 |
| 30 | 50000 | 10000 | 256.07 |
| 40 | 50000 | 10000 | 115.04 |
| 50 | 50000 | 10000 | 39.63 |

从结果看，`obs_mul_factor=30` 在 50000 局训练下效果最好。`obs_mul_factor=40` 和 `50` 理论上提供了更细的状态区分，但在相同训练轮数和探索衰减速度下，效果反而下降。这个现象说明，较大的状态空间需要更多样本和更长探索时间来填充 Q 表，不能只提高离散化精度而不调整训练参数。

基于这个分析，后续对较大的分箱因子进行了参数适配。主要调整为：

1. 将训练轮数从 50000 增加到 100000，使更多状态有机会被访问和更新；
2. 将 $\epsilon$ 指数衰减时间常数从 10000 增加到 20000，延长前中期探索时间；
3. 保持计数学习率衰减 $p=0.4$，让访问次数较少的新状态仍有较大的学习率。

调整后的结果如下：

| 实验 | 训练轮数 | $\epsilon$ 衰减时间常数 | Seed 42 | Seed 100 | Seed 2026 | 最终平均分 |
| :--- | ---: | ---: | ---: | ---: | ---: | ---: |
| `factor_40` | 50000 | 10000 | 53.24 | 71.46 | 220.43 | 115.04 |
| `factor_40_opt` | 100000 | 20000 | 210.24 | 508.85 | 1533.38 | 750.82 |
| `factor_50` | 50000 | 10000 | 23.07 | 54.86 | 40.96 | 39.63 |
| `factor_50_opt` | 100000 | 20000 | 102.56 | 71.44 | 65.89 | 79.96 |

可以看到，`factor_40_opt` 相比未调参的 `factor_40` 有明显提升，最终三种子平均分从 115.04 提高到 750.82。其训练曲线也显示，50k 前得分并不高，但在 70k 以后开始明显提升。这说明 `obs_mul_factor=40` 并不是本身效果差，而是在原来的 50000 局训练和较快探索衰减下没有得到充分学习；当训练时间和探索时间增加后，更细的状态划分可以减少状态混淆，发挥出更高上限。

不过，`factor_50_opt` 的提升有限，最终平均分只有 79.96。这说明分箱继续变细后，状态空间过大，100000 局训练仍然不足以让表格式 Q-learning 充分学习。综合来看，`obs_mul_factor=40` 是当前实验中更合适的状态离散化粒度：它比 30 有更细的状态区分能力，又不像 50 那样导致状态过于稀疏。

## 四、当前超参数选择

根据以上实验，目前采用的主要超参数如下：

| 参数 | 当前选择 | 说明 |
| :--- | :--- | :--- |
| 状态表示 | $(x_{\text{pipe}}, y_{\text{bottom}}, v_y)$ | 使用基础状态表示，`change_obs` 中的线性改动未带来提升 |
| `obs_mul_factor` | 40 | 更细状态划分在延长训练和探索后取得明显提升 |
| 死亡惩罚 | -100 | 中等强度惩罚，避免过大惩罚导致 Q 值剧烈波动 |
| $\gamma$ | 0.97 | 多种子实验中后期表现较平稳 |
| $\epsilon$ 初值 | 0.2 | 训练前期保留足够探索 |
| $\epsilon$ 衰减方式 | 指数衰减 | 固定探索率效果不理想，指数衰减作为后续实验基础 |
| $\epsilon_{\min}$ | 0.01 | 后期保留少量探索 |
| $\epsilon$ 衰减时间常数 | 20000 | 状态空间增大后放慢探索衰减 |
| 学习率方式 | 计数衰减 | 按 `(state, action)` 访问次数调整 |
| 计数衰减幂次 $p$ | 0.4 | 衰减速度实验中末期平均分最高 |
| 训练轮数 | 100000 | `obs_mul_factor=40` 后状态空间增大，需要更多训练样本 |
| 评估局数 | 100 | 用固定评估 seed 计算平均分 |
| 随机种子 | 42、100、2026 | 用于多种子对比 |

对应到当前代码参数，运行前需要先在 `process_obs()` 中将 `obs_mul_factor` 设为 40，训练时主要使用：

```bash
python src/train_ai_or_play.py \
  --train \
  --alpha 0.5 \
  --gamma 0.97 \
  --epsilon 0.2 \
  --decay-method exponential \
  --epsilon-min 0.01 \
  --exp-decay-rate 20000 \
  --alpha-decay-method count \
  --alpha-decay-power 0.4 \
  --iteration 100000 \
  --test-interval 10000 \
  --eval-episodes 100
```

在 `count` 学习率衰减下，实际更新步长由状态-动作访问次数决定：

$$
\alpha_t(s,a)=\frac{1}{N(s,a)^{0.4}}
$$

因此命令中的 `--alpha` 对 Q 值更新影响不大，主要由 `--alpha-decay-power 0.4` 控制学习率变化。
